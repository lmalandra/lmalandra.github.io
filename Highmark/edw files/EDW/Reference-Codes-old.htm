Updated July 2008
The EDW 03 (CEAC03A) code table called EA_CD_CLM_PROC_LOCAL has become a partial replacement for the DB2 code table called DB2PROD.SYS_PROCEDURE_ROOT. This DB2 table is being eliminated by Legacy Mod in the July 2008 release. What I mean by a partial replacement is that the DB2 version had a good number of data elements per a procedure code. Our EDW 03 replacement table will house all the procedure codes, a national BCBS code (can be the same or different), and a short subscriber payable description. You would need to interface with Legacy Mod to get additional procedure code data.

The table CEAC03A.EA_CD_CLM_PROC_LOCAL is now fully populated in production.

Populating the Reference Code Tables
Automated processes are currently being developed. The goals of this project are to:
  • Determine the best single source for code values
  • Build the minimum number of processes that can automate the population and maintenace of the most current RCT data
  • Build a process for monitoring the quality of the RCT data
Each method employs database tables to control the process. With this design we will be able to change how the data is loaded simply by changing the information in these control tables - no programming changes are required. Here are the methods being developed:

Method Description
Corporate Repository There are two main corporate repositories at Highmark.  One is owned by the Business Information Management (BIM) and the other is the Pittsburgh Corporate Code Table system.  This process will allow us to map specific code values to our reference code tables and check each night for changes.

One of the great features of this method is that any business area can work with BIM to create their own set of source data on the corporate repository.  Any changes to the data in the repository will be picked up by the EDW process the same evening.

Development of the process is scheduled to be completed by October 2005.
Corporate Database This process will allow the loading of data directly from DB2 and Oracle corporate databases.   The refreshing of this data can be set to any frequency to accommodate the update schedule for each source.
Text files In this case, any text file can be used to load new RCT data.
