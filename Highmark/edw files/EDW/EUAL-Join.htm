EUAL Join Criteria

Since the EUAL_MAIN table is at the Claim Line level, the most commonly used Claim header related columns are present on each line. This design provides the foundation for more straightforward queries, and simpler grouping functions because all data is at the same level (the Claim Line level).

Supporting information about the Claim is also stored at the Claim Line level in the other EUAL tables (EUAL_DIAG, EUAL_MEDICAL_CD, etc). So, while a join is required to retrieve this additional data, the only data elements required to join are Medical Expense Event ID and Claim Line ID. In addition, there is no need for the end user to know whether the clinical information came from the source system at the Claim level or the Claim Line level, since the clinical subject area data is all stored at the Claim Line level in EUAL.

The primary EAMEE_ID key is maintained from EDW for every row in EUAL. The Line number “blind key” is also preserved from EDW. The blind key Line number column is LINE_BLIND_KEY in all Claim related tables. These primary key values are used for joining tables within the EUAL, as well as joining back to EDW Core and Line tables if required.

The following join SQL details how to join the EUAL Main table to the support Claim Line data tables:
Inner join the EUAL_MAIN to EUAL_DIAG:
EUAL_MAIN.EAMEE_ID = EUAL_DIAG.EAMEE_ID AND
EUAL_MAIN.LINE_BLIND_KEY = EUAL_DIAG.LINE_BLIND_KEY
Inner join the EUAL_MAIN to EUAL_MEDICAL_CD:
EUAL_MAIN.EAMEE_ID = EUAL_MEDICAL_CD.EAMEE_ID AND
EUAL_MAIN.LINE_BLIND_KEY = EUAL_ MEDICAL_CD.LINE_BLIND_KEY
Inner join the EUAL_MAIN to EUAL_PROVIDER:
EUAL_MAIN.EAMEE_ID = EUAL_PROVIDER.EAMEE_ID AND
EUAL_MAIN.LINE_BLIND_KEY = EUAL_PROVIDER.LINE_BLIND_KEY
