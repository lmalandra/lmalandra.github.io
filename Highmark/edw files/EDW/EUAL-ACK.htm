Query by Product, Region or Business Unit using Accounting Control Key

Accounting Control Key (ACK)
The ACK section provides end users with a basic understanding of the components that comprise the ACK, and how the ACK columns available in the EUAL can be utilized to identify claims for Highmark’s Product offerings and/or claims for various Market Segments (Western, Central).

The EUAL has ACK populated for Highmark Western Region, Highmark Central Region, Highmark Plan Partnership, Highmark Subsidiaries Keystone Health Plan West (Western Managed Care), Highmark Health Insurance Co. (Medicare Advantage Freedom Blue West Virginia), and Highmark Senior Services (Medicare Part D PA and WV statewide) and Mountain State Blue Cross of West Virginia.

Note; ACK fields are not populated for the following business; agent plan business where Highmark jointly underwrites traditional and comprehensive coverage with Independence Blue Cross and Blue Cross of NEPA, ITS Blue Card Host and the Federal Employee Program (FEP) for both Highmark and Mt. State.

The ACK code is derived by the ACK Common Module, and all components of ACK are housed in the EUAL_MAIN TABLE. The ACK code components consist of Business Unit, Region Code, Product, Rating Category and Income Control. Following is a description of each component of the ACK:

Using ACK to identify Market Segments
The market segment refers to the geographical area in which Highmark Inc. and its subsidiaries and its affiliate Mt. State BCBS of West Virginia operate as health insurance carriers.

The ACK Business Unit identifies the various legal entities/companies that Highmark Inc. and Mt. State conduct business as within the various markets. The ACK Region Code identifies the geographical areas in which these business units conduct business. Refer to Market Segment Matrix for a cross-reference of ACK Business Unit and Region Code to Market Segment.

The ACK Business Unit and Region Code can be used to identify claims for specific market segments. For example, ACK Region can be used to isolate Western Region claims by applying the following where criteria:
Where ACKR_CD in ( ‘PA03’,’PA99’,’RGP3’)
This would return all claim utilization for members enrolled with Highmark, but please note it would also include HMO members enrolled with Keystone Health Plan West (KHPW) and Medicare Part D members enrolled with High Senior Resources (HSR).

If KHPW claims are not wanted, the business unit would be used in the where criteria as follows:
Where ACKR_CD in ( ‘PA03’,’PA99’,’RGP3’)
And ACKBU_CD <> ‘KHPW’

SQL EXAMPLES UTILIZING PRODUCT HIERARCHY TABLE:

The following example extracts all HMO product offerings for the Pa Western Region:
SELECT EUAL_MAIN.*

FROM ceav000a.EUAL_MAIN AS MAIN
JOIN ceav03a.PSPRT PSPROD
ON MAIN.ACKP_CD = PSPROD.PSPRT_LF_VAL_CD

WHERE MEXCED_LD_CYC_DT BETWEEN '2008-01-01' AND '2008-01-31'
And ACKR_CD in ( 'PA03','PA99','RGP3')
And PSPRT_ND_LVL3_CD = 'HMO'

The following example extracts all HMO product offerings excluding SECURITY BLUE for the Pa Western Region:
SELECT EUAL_MAIN.*

FROM ceav000a.EUAL_MAIN AS MAIN
JOIN ceav03a.PSPRT PSPROD
ON MAIN.ACKP_CD = PSPROD.PSPRT_LF_VAL_CD

WHERE MEXCED_LD_CYC_DT BETWEEN '2008-01-01' AND '2008-01-31'
And ACKR_CD in ('PA03','PA99','RGP3')
And (PSPRT_ND_LVL3_CD = 'HMO'
And PSPRT_ND_LVL4_CD <> 'HMOSECBL')

The following example extracts all PPO product offerings for the Pa Central Region:
SELECT EUAL_MAIN.*

FROM ceav000a.EUAL_MAIN AS MAIN
JOIN ceav03a.PSPRT PSPROD
ON MAIN.ACKP_CD = PSPROD.PSPRT_LF_VAL_CD

WHERE MEXCED_LD_CYC_DT BETWEEN '2008-01-01' AND '2008-01-31'
And ACKR_CD = 'PA05'
And PSPRT_ND_LVL3_CD = 'PPO'
Note 1: There is a small percentage (less than 0.2%) of Western and Central claims that do not have an ACK Product Code or ACK Region Code. For most analyses, this should not be an issue. However, for individual who are interested in pulling all possible claims for Western and Central business, additional fields can be queried, such as the Enrollment Source Code (EAES_CD) and the Reimbursement Plan Identifier (REIMM_PLN_ID).
