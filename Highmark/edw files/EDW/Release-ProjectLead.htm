EDW Project Lead and Team Resources

Project Lead Responsibilities
This outline defines the high-level responsibilities of the EDW ‘Project Lead’ Role.

Project Timeline and Work Effort template
The “Project Template” is used by the project lead to report project estimates regarding timeline, estimated work effort hours and team status. See the Project Template Reporting Instructions for additional details on when and how to complete and where to submit.

EDW System Defect Process – Quick Reference Guide
This quick reference guide provides high-level guidance on the following topics:
  • Creating a new System Defect
  • Workflow tasks of an SD
  • Closing a System Defect
The Impact Analysis Template is to be completed and inserted in the System Defect at time of creation.

EDW Migration Cross-Reference Template
The “Migration Cross-Reference template” is used by the project lead to identify key pieces of information (such as job names, package ID's, contracts, etc.) that are required for a release.  See the Migration Cross-Reference Template Instructions for additional details of when and how to complete and where to submit. 

Invite List for Walkthroughs and Required Sign-offs
Each project requires a walkthrough for each 'HPLC' project component including Project Initiation, Detailed Requirements, Preliminary Design, Detailed Design and Test Strategy. The following indicate who is required to be invited to the walkthroughs and who is required to sign-off from the EDW team.
  • Compatibility Releases and Non-Critical SDs
  • Strategic Initiative Projects and Critical SDs

EDW HPLC Resources
The EDW uses the HPLC methodology for all projects.  To accommodate the wide variation in size and complexity, there are three levels of HPLC documentation.  Below are links to EDW Standard Operating Procedure documents and templates.
  • EDW HPLC Standard Operating Procedure
  • Highmark Project Lifecycle Information Center > 900 hours
  • HPLC One-Doc Template > 300 hours
  • HPLC Lite Standard Operating Procedure 300 hours or less

Detailed Implementation Plan - Non Corporate Releases
The attached policy (effective October 2007) is to be followed for all EDW non-corporate releases. It outlines responsibilities regarding the release dates and detailed implementation plans for each, as well as required approvals. A customizable excel version of a Detailed Implementation Plan Template is to be completed and saved in the EDW-Non corporate Project Specific Release folder.

EDW Team Reports
SD Maintenance
This report summarizes the SD activity resulting from the weekly SD Maintenance and Review meeting where SD's are added and removed from EDW releases.

Request for Release Change
This form is to be completed and submitted to EDW-FIN-EPM in order to satisfy the request for a release date change. It is important to understand, by completing the attached form, you will be providing the team with the necessary documentation to address your request in a timely manner.
